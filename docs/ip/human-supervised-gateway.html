Human-Supervised Gateway for Persistent AI

Public-safe abstraction: This note establishes prior art around persistent agents operating behind a human-supervised gateway. It does not reveal gateway internals, tokens, session mechanics, edge cases, production routing, or security-sensitive deployment details.

Thesis

Persistent AI systems need a gateway layer that separates observation, drafting, reversible action, external communication, and privileged operations.

The gateway is not just an API proxy. It is a supervision boundary.

Core pattern

A human-supervised gateway can provide:

  • Identity-bound sessions: agents operate in controlled sessions.
  • Tool mediation: tools are declared and routed through the gateway.
  • Permission classes: different actions require different levels of approval.
  • Audit trail: actions and approvals are recorded.
  • Recovery support: agents can resume after interruption.
  • Delivery control: user-facing messages and external sends can be gated.

Permission classes

A safe public model may include read-only inspection, local draft creation, reversible local edits, external messages, financial or legal actions, destructive operations, and privileged system changes. Each class can have its own approval rule.

Why this matters

Without a gateway, persistent agents become either too weak to be useful or too risky to trust. A supervision gateway lets the system support autonomy while keeping humans in control of high-impact actions.

Defensive claim: A human-supervised gateway for persistent AI agents can mediate tools, sessions, permission classes, audit trails, delivery control, and recovery support.

What this note intentionally does not reveal

Safe toy example

An agent may summarize a local file without approval, draft a reply with approval required before sending, and refuse destructive file deletion unless explicitly confirmed.

Closing

Autonomy without supervision is unsafe. Supervision without continuity is weak. A human-supervised gateway is the runtime boundary that makes persistent agents practical.
